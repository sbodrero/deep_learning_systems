1. Report Overview

This report documents a controlled deep learning experiment for image classification using the Fashion-MNIST dataset — 70,000 grayscale 28×28 images of clothing items across 10 categories. Two Convolutional Neural Network (CNN) configurations were implemented in PyTorch and trained for 10 epochs each: a baseline SimpleCNN with no regularization, and an experimental variant adding a single Dropout layer (p = 0.5) to the classifier. The experiment was designed to isolate the effect of dropout regularization on generalization performance. Both models were evaluated on the held-out 10,000-image test set, and the experimental model achieved 92.85% test accuracy versus 91.55% for the baseline — a +1.30 percentage point improvement that demonstrates dropout's effectiveness in reducing mild overfitting even in a shallow CNN. The dataset is publicly available via torchvision.datasets.FashionMNIST and is reproduced from Xiao et al. (2017) (see reference 4).

2. Dataset and Task Description

The Fashion-MNIST dataset was created by Zalando Research as a more challenging drop-in replacement for the original MNIST handwritten digit dataset. It contains 70,000 grayscale images (60,000 training, 10,000 test) of size 28×28 pixels, each labelled with one of 10 clothing categories:

The task is 10-class image classification — a supervised deep learning task appropriate for Convolutional Neural Networks. CNNs are the canonical choice because they exploit the spatial structure of image data through learned local filters, sharing weights across spatial positions to capture translation-invariant features efficiently (LeCun et al., 1998). The dataset is perfectly balanced (each class contains exactly 6,000 training and 1,000 test images), which means accuracy is an appropriate primary metric alongside per-class analysis.

Preprocessing: Images were normalized using the Fashion-MNIST population statistics (mean = 0.2860, std = 0.3530), converting pixel values from [0, 1] to a zero-centred distribution. Normalization stabilizes gradient magnitudes during backpropagation and improves convergence speed. The dataset was not augmented (random flips, crops, etc.) in this baseline experiment — augmentation is identified as a future improvement in Section 8.

Dataset access: The dataset is automatically downloaded by torchvision.datasets.FashionMNIST(root='./data', download=True) on first run. No manual download is required.

3. Model Architecture and Design Decisions

The baseline SimpleCNN follows the standard convolutional feature extractor + linear classifier pattern:

Design rationale:

  • Two convolutional blocks: 28×28 images after two 2×2 max-pool operations have 7×7 spatial extent. A third pooling stage would reduce this to 3×3 — too small for the filters to capture meaningful structure. Two blocks are therefore appropriate for this input size.
  • Filter counts 32→64: A common doubling pattern that increases representational capacity at each block while the spatial footprint shrinks. This balances depth and parameter efficiency.
  • ReLU activations: Preferred over sigmoid or tanh because they avoid vanishing gradients for positive activations and are computationally inexpensive.
  • Adam optimizer (lr=0.001): Adam adapts learning rates per parameter using first and second moment estimates, making it robust to hyperparameter choices and widely used as a default for deep learning tasks.
  • CrossEntropyLoss: The standard loss for multi-class classification with integer class labels. It combines a log-softmax activation with negative log-likelihood, numerically stable and appropriate for the 10-class output.

Why dropout was selected as the experimental variable: Observing the baseline training logs revealed a diagnostic pattern: training loss continued to decrease steeply (0.4009 → 0.0535 over 10 epochs) while test accuracy plateaued between epochs 6–10 (91.53%–91.94%). This divergence between training loss and test accuracy is a classic signature of mild overfitting — the model memorizes training-specific patterns rather than generalizing. Dropout is the most direct intervention for this problem: during each forward pass in training, it randomly zeros 50% of FC1 activations, preventing neurons from co-adapting and forcing the network to learn more distributed, redundant representations (Srivastava et al., 2014). At inference time, dropout is disabled and all activations are scaled by (1 − p) to preserve expected output magnitude. The change is minimal and controlled — exactly one component differs between the two configurations.

5.1 Training Dynamics

The baseline training loss fell sharply from 0.4009 (epoch 1) to 0.0535 (epoch 10), while test accuracy grew from 89.24% to 91.55% but effectively plateaued after epoch 6. The gap between the near-zero training loss and the 91% test accuracy confirms mild overfitting: the model fits the training distribution almost perfectly but generalizes less effectively to unseen images.

The CNN + Dropout training loss remained higher throughout (0.4752 → 0.1358), which is expected — dropout introduces stochasticity during training that prevents the training loss from reaching the same minimum as the unregularized model. However, test accuracy improved more steadily and reached 92.85%, demonstrating that the model generalizes better despite (or because of) the higher training loss. This is the textbook effect of dropout regularization: sacrificing training-set fit to gain generalization (Srivastava et al., 2014).

The dropout model shows substantial gains on T-shirt/top (+10.1 pp), Pullover (+4.8 pp), and Sneaker (+3.4 pp) — all classes that were difficult for the baseline. These improvements suggest that the baseline was memorizing specific training-set patterns for these classes rather than learning general features. The dropout model shows slight regressions on Coat (−4.6 pp) and Ankle boot (−1.7 pp), which may reflect that these classes required more precise feature memorization that dropout partially disrupts.

5.3 Hardest Classes — A Concrete Error Case

The Shirt class achieved only 77.7% (baseline) and 78.5% (dropout) — by far the worst per-class accuracy. Inspection of the confusion matrix reveals that Shirts are most often misclassified as T-shirts/tops, Coats, and Pullovers. This is a genuine challenge: at 28×28 grayscale resolution, a shirt and a T-shirt share very similar collar and sleeve structure. The visual difference — typically button placket, collar type, or fabric texture — is largely invisible at this resolution and in the absence of colour information. This is not a failure of the model design; it reflects a fundamental limitation of the input representation, as discussed in Section 6.

6. Limitations and Risks

No validation split — test set used for monitoring: Training loss and test accuracy were tracked together at every epoch, effectively using the test set as a proxy for validation. Under strict machine learning discipline, the test set should be evaluated only once — at the end of training — after hyperparameter selection is complete. In this experiment, the test accuracy curve was visible during training, which could introduce implicit selection bias (e.g., stopping at 10 epochs partly because that is where test accuracy is satisfactory). A proper train/validation/test split (e.g., 80/10/10) should be used in future work.

Low-resolution grayscale input: 28×28 pixels with no colour information is severely limited for some categories. Shirt vs. T-shirt, Coat vs. Pullover, and Sneaker vs. Ankle boot are regularly confused by both models. This ambiguity is inherent to the representation, not resolvable by model architecture changes alone.

No data augmentation: The training set is used as-is. Random horizontal flips, small rotations, and random cropping — standard augmentation strategies — were not applied. These would increase effective training set diversity and likely further reduce overfitting, particularly for symmetric items like bags and shoes.

Single random seed: Results are reported for a single training run with torch.manual_seed(42). The +1.30 pp improvement of CNN + Dropout over baseline may vary across random initializations. Multiple runs with averaged results would give a statistically reliable estimate of the effect size.

7. Ethical and Responsible Use

Potential misuse — surveillance and consumer profiling: A model trained to classify clothing items from images could be applied to surveillance footage or social media images to profile individuals by clothing style, infer socioeconomic status, or track individuals across locations without consent. Fashion-MNIST is a clean benchmark dataset with no personal information, but the architectural pattern (CNN applied to clothing images) generalises directly to such applications. Deploying such a model in any context where individuals have not consented to having their clothing analysed would raise serious privacy and civil liberties concerns.

Bias in representation: Fashion-MNIST was curated from Zalando's European e-commerce catalogue. The clothing styles, cuts, and visual presentation reflect a specific geographic and commercial context. A model trained on this data would likely perform poorly on clothing from other cultural contexts (e.g., traditional garments from South or East Asia, African fashion). Deploying such a model as a general clothing classifier without awareness of this distributional bias could produce systematically inaccurate results for underrepresented styles.

Step proposed to reduce risk: Before any real-world deployment of a clothing classification model, a bias audit should be conducted on a geographically and culturally diverse test set. Model confidence scores (softmax probabilities) should be exposed to downstream systems so that uncertain predictions can be flagged for human review rather than acted upon automatically.

8. Future Improvements

  • Data augmentation: Random horizontal flips and small random crops would increase the effective diversity of the training set and are expected to further reduce overfitting, particularly for symmetric classes (bags, shoes).
  • Proper train/validation/test split: Reserve 10% of the training set as a validation set for hyperparameter selection. Evaluate on the test set only once at the end of the full experiment.
  • Batch normalization: Adding nn.BatchNorm2d after each convolutional activation would normalise feature maps across the batch, stabilise training gradients, and potentially allow a higher learning rate — often yielding faster convergence and better final accuracy than dropout alone.
  • Deeper architecture: A third convolutional block (64→128 channels) with a corresponding FC layer increase may capture higher-level features that help distinguish difficult class pairs (Shirt vs. T-shirt).
  • Multiple training runs: Report mean and standard deviation of test accuracy across 5 seeds to provide statistically reliable comparison between baseline and experimental configurations.
  • Class-weighted loss: Although Fashion-MNIST is balanced overall, the Shirt class may benefit from a higher loss weight to increase model sensitivity to this difficult category.
